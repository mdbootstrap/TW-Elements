Prerequisites

Before starting the project make sure to install the following utilities:

  • Node LTS (18.x.x or higher recommended)
  • Code editor. We recommend VSCode

Creating a new Next application

Let's create a fresh Next application so that we can go through all the steps together.

Step 1

Create new project.

Note: Options used while creating the app: Typescript - Yes, ESLint - Yes, `src/` directory - Yes, App Router - both examples included in this tutorial.

If you encounter the Request is not defined error, consider updating your Node.js version to at least v18.0.0 or higher. Alternatively, you can use Next.js v13.0.0 or a more recent version to address this issue.

{{< twsnippet/wrapper "Terminal" "" "mobile" >}} {{< twsnippet/code active=true lang="plaintext" type="terminal" >}} npx create-next-app@latest cd my-project {{< /twsnippet/code >}} {{< /twsnippet/wrapper >}}

Step 2

Run the development server.

{{< twsnippet/wrapper "Terminal" "" "mobile" >}} {{< twsnippet/code active=true lang="plaintext" type="terminal" >}} npm run dev {{< /twsnippet/code >}} {{< /twsnippet/wrapper >}}

Add the paths to all of your template files in your tailwind.config.js file.

Add the @tailwind directives for each of Tailwind's layers to your globals.css. Additionally, delete the default font styles in layout.tsx file.

Troubleshooting

Problem overview:

Users might face difficulties with improper functioning of components in some frameworks for TW Elements. This is due to the fact that some of them aren't initializing properly.

Suggested solution:

Consider importing tw-elements package dynamically. By doing it, you can enhance the initialization process, potentially resolving any issues related to component functionality. Additionally, ensure that the initTE() method is called in the correct sequence and with the appropriate parameters, facilitating a smoother experience with TWE components.

Note: It should also solve errors like document is not defined or (Name of TWE Component) is not imported from "tw-elements". For the second issue, attempting to import components in ES format could be an alternative solution.

ES import:

{{< twsnippet/wrapper "with next.config.js configuration, without next.config.js configuration" "" "mobile" >}} {{< twsnippet/code active=true lang="JavaScript" >}} // next.config.js module.exports = { webpack: (config, { isServer }) => { // For packages that provide different entry points, specify the one you want to use. config.resolve.alias = { ...config.resolve.alias, "tw-elements": "tw-elements/dist/js/tw-elements.es.min.js", }; return config; }, }; // component.tsx useEffect(() => { import { Tooltip, initTE } from "tw-elements"; initTE({ Tooltip };) }, []); {{< /twsnippet/code >}} {{< twsnippet/code lang="JavaScript" >}} useEffect(() => { import { Tooltip, initTE } from "tw-elements/dist/js/tw-elements.es.min.js"; initTE({ Tooltip };) }, []); {{< /twsnippet/code >}} {{< /twsnippet/wrapper >}}

ES Dynamic import:

{{< twsnippet/wrapper "Code" "" "mobile" >}} {{< twsnippet/code active=true lang="JavaScript" >}} useEffect(() => { const init = async () => { const { Tooltip, initTE } = await import("tw-elements"); initTE({ Tooltip }); }; init(); }, []); {{< /twsnippet/code >}} {{< /twsnippet/wrapper >}}

Problem overview:

Users may encounter initialization issues with TWE components when navigating through applications with routing mechanisms. The default behavior of the initTE method includes checking if components are already initialized, which can lead to complications when components are unmounted and then remounted during routing.

Suggested solution:

Consider using using autoReinits option in initTE method. By default it's value is set to false. By changing the value to true, the initTE method won't check if components were already initialized.

{{< twsnippet/wrapper "Code" "" "mobile" >}} {{< twsnippet/code active=true lang="JavaScript" >}} initTE({ Tooltip }, { allowReinits: true }); {{< /twsnippet/code >}} {{< /twsnippet/wrapper >}}
