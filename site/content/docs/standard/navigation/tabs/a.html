Import

Importing components depends on how your application works. If you intend to use the TW elements ES format, you must first import the component and then initialize it with the initTE method. If you are going to use the UMD format, just import the tw-elements package.

{{< twsnippet/no-demo id="api-example41" >}}
{{< twsnippet/wrapper "javascript,umd" >}} {{< twsnippet/code active=true lang="js" >}} import { Tab, initTE } from "tw-elements"; initTE({ Tab }); {{< /twsnippet/code >}} {{< twsnippet/code lang="js" >}} import "tw-elements"; {{< /twsnippet/code >}} {{< /twsnippet/wrapper >}}

Usage

Regarding accessibility

If you’re using navs to provide a navigation bar, be sure to add a role="navigation" to the most logical parent container of the <ul>, or wrap a <nav> element around the whole navigation. Do not add the role to the <ul> itself, as this would prevent it from being announced as an actual list by assistive technologies.


Note that navigation bars should not be given role="tablist", role="tab" or role="tabpanel" attributes. These are only appropriate for dynamic tabbed interfaces, as described in the WAI ARIA Authoring Practices. The aria-current attribute is not necessary on dynamic tabbed interfaces since our JavaScript handles the selected state by adding aria-selected="true" on the active tab.

Via data attributes

You can activate a tab or pill navigation by simply specifying data-te-toggle="tab" or data-te-toggle="pill" on an element.

Classes

Custom classes can be passed via data attributes or JavaScript. For data attributes, append the class name to data-te-class, as in data-te-class-show="opacity-100 bg-red-600"

Name Default Description
hide opacity-0 Sets styles for hidden tab.
show opacity-100 Sets styles for visible tab.
{{< twsnippet/no-demo id="api-example8" >}}
{{< twsnippet/wrapper "javascript,umd" >}} {{< twsnippet/code active=true lang="js" >}} const someTabTriggerEl = document.querySelector('#someTabTrigger'); const tab = new Tab(someTabTriggerEl, {show: "opacity-100 bg-red-600"}); {{< /twsnippet/code >}} {{< twsnippet/code lang="js" >}} const someTabTriggerEl = document.querySelector('#someTabTrigger'); const tab = new te.Tab(someTabTriggerEl, {show: "opacity-100 bg-red-600"}); {{< /twsnippet/code >}} {{< /twsnippet/wrapper >}}

Methods

Asynchronous methods and transitions:
All API methods are asynchronous and start a transition. They return to the caller as soon as the transition is started but before it ends. In addition, a method call on a transitioning component will be ignored.

Method Description Example
show Selects the given tab and shows its associated pane. Any other tab that was previously selected becomes unselected and its associated pane is hidden. Returns to the caller before the tab pane has actually been shown (i.e. before the shown.te.tab event occurs). instance.show()
dispose Destroys an element’s tab. instance.hide()
getInstance Static method which allows you to get the tooltip instance associated with a DOM element. Tab.getInstance(someTabTriggerEl)
{{< twsnippet/no-demo id="api-example6" >}}
{{< twsnippet/wrapper "javascript,umd" >}} {{< twsnippet/code active=true lang="js" >}} const someTabTriggerEl = document.querySelector('#someTabTrigger'); const tab = new Tab(someTabTriggerEl); tab.show(); {{< /twsnippet/code >}} {{< twsnippet/code lang="js" >}} const someTabTriggerEl = document.querySelector('#someTabTrigger'); const tab = new te.Tab(someTabTriggerEl); tab.show(); {{< /twsnippet/code >}} {{< /twsnippet/wrapper >}}

Events

When showing a new tab, the events fire in the following order:


  • hide.te.tab (on the current active tab)
  • show.te.tab (on the to-be-shown tab)
  • hidden.te.tab (on the previous active tab, the same one as for the hide.te.tab event)
  • shown.te.tab (on the newly-active just-shown tab, the same one as for the show.te.tab event)

If no tab was already active, then the hide.te.tab and hidden.te.tab events will not be fired.

Event type Description
show.te.tab This event fires on tab show, but before the new tab has been shown. Use event.target and event.relatedTarget to target the active tab and the previous active tab (if available) respectively.
shown.te.tab This event is fired when the tooltip has been made visible to the user (will wait for CSS transitions to complete).
hide.te.tab This event is fired immediately when the hide instance method has been called.
hidden.te.tab This event is fired when the tooltip has finished being hidden from the user (will wait for CSS transitions to complete).
inserted.te.tab This event is fired after the show.te.tab event when the tooltip template has been added to the DOM.
{{< twsnippet/no-demo id="api-example7" >}}
{{< twsnippet/wrapper "JavaScript" >}} {{< twsnippet/code active=true lang="js" >}} const tabEl = document.querySelector('a[data-te-toggle="pill"]'); tabEl.addEventListener('shown.te.tab', (e) => { e.target // newly activated tab e.relatedTarget // previous active tab }); {{< /twsnippet/code >}} {{< /twsnippet/wrapper >}}
